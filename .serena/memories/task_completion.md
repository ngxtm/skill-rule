# Task Completion Checklist

## Before Committing
1. **Build**: `bun run build` - Ensure no compile errors
2. **Format**: `bun run format` - Format code with Prettier
3. **Lint**: `bun run lint` - Check for linting issues
4. **Test**: `bun test` - Run test suite

## Commit Guidelines
- Use conventional commit format
- Keep commits focused
- No confidential info in commits

## After Implementation
1. Run build to verify compilation
2. Test CLI commands manually if needed
3. Update README if adding new features
4. Bump version in package.json if releasing
